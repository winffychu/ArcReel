# Agent 故障观测保留证据，不推断根因

用户反馈中的 Agent 启动错误可能因空异常消息而丢失异常类型，SDK 合成的轮次错误也曾因结构化标记在事件日志写入点丢失而被渲染成助手回答。决定让 Agent 启动失败与轮次失败共用“最小通用外壳 + 脱敏原始载荷”的故障观测契约：外壳只承载 ArcReel 确知的阶段、时间、项目和会话上下文，原始载荷保留实际获得的异常链、stderr、SDK 消息 / 结果及上游返回；不建立上游错误类型枚举，也不把上游解释升级为 ArcReel 的根因判断。

轮次失败在会话事件日志写入点定型为系统故障事件，不再写成助手消息；启动失败通过结构化 API 错误即时返回，并连同轮次失败写入现有诊断日志。轮次失败沿用会话事件日志持久化，启动失败不跨刷新保留；不新增故障表、专用文件或历史查询接口，也不主动迁移或重写已经物化的历史错误事件。

原始载荷只覆盖实际收到或生成的故障对象，不主动复制提示词、正常流式输出或完整 transcript；载荷本身携带的上下文仍予以保留。仅遮蔽可直接冒用身份或产生扣费的秘密值，例如 API key、认证令牌、Cookie、密码和签名 URL 的敏感参数；其余路径、异常类型、原因链、堆栈与未知字段不截断、不翻译、不润色。

界面共用两层故障卡片：默认层显示阶段、观测到的类型 / 状态和有来源标注的原始消息，并明确这些事实不等同于根因；展开层呈现并可复制完整脱敏载荷。只提供与根因无关的操作——复制详情、打开 Agent 设置，以及对启动失败重新尝试；诊断日志仍由现有系统设置入口下载，不在故障卡片重复提供。轮次失败不自动重放原消息，避免重复执行已经发生的工具操作。

## 明确不采用

- 根据错误文案归类根因或生成针对性修复建议。
- 为未知的上游错误维护穷举映射或降级为无信息的“未知错误”。
- 为低价值的历史失败轮次引入 transcript 关联回填。
- 自动重放失败轮次。
